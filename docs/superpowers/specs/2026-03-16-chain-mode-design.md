# 不斷鏈模式（Chain Mode）設計文件

**Date:** 2026-03-16

**Problem:** 現有的連續天數（streak）累積對用戶缺乏約束力——中斷沒有真實後果，只是數字歸零，心理代價太低。

**Solution:** 在習慣層級加入可選的「不斷鏈模式」，啟用後若有一天未完成，該習慣的所有累積基金金額清零，並顯示有儀式感的悼念畫面，利用損失厭惡心理強化行為約束。

---

## 核心規則

- **手動opt-in**：每個習慣可獨立啟用，非強制全套
- **僅限每日習慣**：頻率為每日的習慣才適用
- **清零範圍**：streak 數字 + 這個習慣累積的所有基金金額
- **中斷偵測**：App 載入時自動偵測前一天是否有打卡
- **中斷提示**：戲劇感悼念畫面，讓用戶感受損失

---

## 資料模型

Habits 表新增三個欄位：

| 欄位 | 類型 | 預設值 | 說明 |
|------|------|--------|------|
| `chain_mode` | `boolean` | `false` | 是否啟用不斷鏈模式 |
| `chain_broken_at` | `date` | `null` | 最近一次斷鏈的日期（null = 從未斷） |
| `chain_peak_streak` | `integer` | `0` | 斷鏈前的最高連續天數（悼念畫面顯示用） |

### 基金計算邏輯

- `chain_mode = false`：照舊計算所有 checkin 的累積金額
- `chain_mode = true`：只計算 `chain_broken_at` 之後的 checkin（若曾斷過）；等同清零效果，但歷史數據不破壞性刪除

---

## 斷鏈偵測流程

**觸發時機：** App 載入時（`useEffect` on mount），在撈取習慣資料後執行

**步驟：**
1. 篩出所有 `chain_mode = true` 的習慣
2. 對每個習慣，檢查昨天（`todayStr() - 1`）是否有 checkin 記錄
3. 若昨天無 checkin，且習慣建立日期 ≤ 昨天（排除剛建立當天的習慣）：
   - 計算「目前這輪連續天數」（從上次 `chain_broken_at` 之後算起）
   - 計算「這個習慣目前累積的基金金額」
   - 更新 DB：`chain_broken_at = 昨天`，`chain_peak_streak = 計算出的天數`
   - 將悼念資料推入佇列：`{ habitName, peakStreak, lostAmount }`
4. 若佇列非空，依序顯示悼念畫面

---

## 悼念畫面

全螢幕覆蓋（`position: fixed, inset: 0`），深色背景（`#1A1714`），置中內容。

若同時有多個習慣斷鏈，逐一顯示，按「我知道了，重新開始」後才顯示下一個。

**畫面內容：**

```
😢

哭哭

「{習慣名稱}」
習慣中斷啦

你曾連續堅持了  {peakStreak} 天
NT${lostAmount} 的累積已歸零

―

損失不可挽回。
但今天可以重新開始。

[ 我知道了，重新開始 ]
```

**樣式方向：**
- 背景：`#1A1714`（深棕黑）
- 文字：白色
- emoji 大字顯示（`font-size: 64px`）
- 標題「哭哭」：Georgia serif，大字
- 習慣名稱：引號包覆，muted 色
- 天數 / 金額：突出顯示（大字、白色）
- 按鈕：warm 色（`#E76F51`），圓角，全寬

---

## 啟用方式

**位置：** Habits tab 的習慣卡片

- `chain_mode = false`：卡片顯示小按鈕「🔗 開啟不斷鏈」
- `chain_mode = true`：卡片顯示標籤「⛓️ 不斷鏈中」

**啟用確認提示（modal bottom sheet）：**

```
你確定要啟用「不斷鏈模式」嗎？

一旦啟用，只要有一天沒完成，
這個習慣的所有累積基金將歸零。

[ 取消 ]   [ 啟用，我做得到 ]
```

**停用：** 點擊「⛓️ 不斷鏈中」可停用，停用不清零，只是不再偵測中斷。

---

## Out of Scope（本次）

- 週 X 次頻率的習慣不支援不斷鏈模式
- 不支援「補救機制」（中斷後付罰金保留累積）
- 不支援 push notification 提醒快要斷鏈
- 不支援「護盾」道具（允許偶爾中斷一次）
